Look Mum, no VM!

Finding zen in my Windows life using the Linux subsystem

Kate Casey
Technical Lead

github.com/katecasey/devnq_lightning

What is all this Linux subsystem business about anyway?

  • Linux-compatible kernel interface; no actual Linux kernel code
  • Supplementary dev env for Windows users:
    • GNU user space
    • Shell and command language
    • Native command line tools
    • Programming language interpreters
    • Graphical applications via X11
  • Flavours include Ubuntu, SUSE, Debian, and more

Why did I decide to give it a go?

  • Things I want/need to do that Linux does more gracefully out-of-the-box
  • Can't switch fully to Linux; enterprise, Windows-hugging applications
  • 2 disk partitions and native Linux install is inflexible and slow
  • Local VMs can memory-hungry and complex for file sharing

How did I do it?

  1. Windows setup prerequisites
  2. Microsoft store and installation
  3. Bootstrapping (first update, inbuilt tools, must-have third-party installs)
    • bash, tmux, vim, pip, ipython, git, dig, grep, ssh-keygen, sudoers insults...

What's fabulous?

  • C:\ mount point
  • All your favourite packages; Git without drama, file analysis tools
  • The power of piping
  • GUI apps if needed and supported
  • Memory usage + benchmarking is very close to native Linux except for some I/O operations

What pitalls should you be aware of?

  • Windows and Linux files don't always play nicely
  • Not all Linux applications can be run (32-bit binaries, kernel modules, device drivers)
  • X11 support not exhaustive, issues with hardware acceleration, funky screen artefacts
  • No audio support

But what does it all mean?

  • My life is at least 100% more zen than it was before
  • My work is possible, whilst my dev environment is fully extensible with all the tools I need
  • This presentation was brought to you by my Linux subsystem
  • And best of all, by the laws of transference, my Windows box also has Super Cow Powers
