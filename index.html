What is all this Linux subsystem business about anyway?

  • Linux-compatible kernel interface; no actual Linux kernel code
  • Supplementary dev env for Windows users:
    • GNU user space
    • Shell and command language
    • Native command line tools
    • Programming language interpreters
    • Graphical applications via X11
  • Flavours include Ubuntu, SUSE, Debian, and more

Why did I decide to give it a go?

  • Things I need to do that Linux does more gracefully out-of-the-box
  • Wish list - bash, tmux, vim, pip, ipython, git, dig, grep, ssh-keygen...
  • Can't switch fully to Linux; enterprise, Windows-hugging applications
  • 2 disk partitions and native Linux install is inflexible and slow
  • Local VMs are memory-hungry and messy
